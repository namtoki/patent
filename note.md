# Note

## 発明とは

自然法則を利用した技術的思想の創作のうち高度なもの

発明であるか + 記載要件 + 実体的要件 (新規性, 進歩性)

### 種類
- ものの発明
- 方法の発明
  - 方法の発明の構成要素は、「すること」という処理または操作の組み合わせ

### 条件
- 課題
- 効果
- 自然法則
  - インターネットによる情報入手やコンピュータによる情報の比較は、光信号や電流による作用という自然法則を利用しているので、発明と認められる
- 新規性
  - 構成要素
  - 構成要素どうしの関係
    - i.e.
      - スプレーのついている位置
      - ローラの取り付け角度
    - Pattern
      - 付加
      - 置換
      - 除去
      - 形態変更
      - 数量変更、分割
      - 共通化、合体

### 記載要件

#### 出願書類
- 願書
- 特許請求の範囲
- 明細書
  - 発明の名称
  - 図面の簡単な説明
  - 発明の詳細な説明
    - 技術分野
    - 従来の技術
    - 発明が解決しようとする課題
    - 課題を解決するための手段
    - 発明の効果
    - 図面の簡単な説明
    - 発明を実施するための形態
- 要約書
- 図面

#### 要件
- サポート要件
- 当事者実施要件
- 明確性要件
- 簡潔性要件
- 単一性要件

### 実体的要件
- 新規性
  - 否定要因
    - 組み合わせがすでに知られている
    - 要素どうしの関係に違いが無い
    - 組み合わせやその要素の概念が重複している
- 進歩性 (特許出願時に世界基準で知られていた技術に基づいて、当業者が簡単にその発明を思いつくかどうか)
  - 否定要因
    - 組み合わせが論理的に導ける (いずれかの要素の発明に示唆されている)
    - 要素どうしの技術分野の関連性が高い
    - 要素どうしの課題の共通性が高い
    - 要素どうしの効果の共通性が高い
    - ただの設計変更
    - ただの寄せ集め (組み合わせが新たな機能的な関係を生じない)

### 図5.1 先行技術調査を受けての対応の流れ

```mermaid
flowchart TD
    S1["<ステップ1><br>構成要素の確認"]
    S2["<ステップ2><br>要素を開示する先行技術の調査"]
    S3{"<ステップ3><br>すべての要素が<br>一つの先行技術に開示<br>されているか？"}
    S4["<ステップ4><br>発明を再検討"]
    S5{"<ステップ5><br>先行技術に<br>開示されてない<br>要素があるか？"}
    S6{"<ステップ6><br>進歩性の<br>否定因子があるか？"}
    S7{"<ステップ7><br>進歩性の<br>肯定因子があるか？"}
    S8{"<ステップ8><br>組み合わせ<br>による技術的効果<br>があるか？"}

    NoNovelty["新規性がない発明"]
    Ready["出願できる状態"]
    Combine["先行技術を組み合わせて<br>発明に至るかの検討"]
    OK1["出願できる"]
    OK2["出願できる"]
    NG1["再検討"]
    NG2["再検討"]

    S1 --> S2
    S2 --> S3
    S3 -->|YES| NoNovelty
    NoNovelty --> S4
    S4 --> S3
    S3 -->|NO| S5
    S5 -->|YES| Ready
    Ready --> Combine
    S5 -->|NO| Combine
    Combine --> S6
    S6 -->|YES| S7
    S6 -->|NO| S8
    S7 -->|YES| OK1
    S7 -->|NO| NG1
    S8 -->|YES| OK2
    S8 -->|NO| NG2
```
