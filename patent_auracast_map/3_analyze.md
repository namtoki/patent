## 1. 提案発明の概要

### 発明の名称（案）
Auracastブロードキャスト情報の収集・蓄積システムおよび地図表示方法

### 発明の構成要素

| 構成要素 | 説明 |
|----------|------|
| A. クライアント端末 | Auracast受信可能なスマートフォン/デバイス |
| B. Auracast情報収集部 | 端末がAuracast送信機を検出し情報を収集 |
| C. サーバシステム | ブロードキャスト情報を蓄積・管理するクラウドサーバ |
| D. データベース | 位置情報とAuracastメタデータを格納 |
| E. 地図表示アプリ | 地図UI上にAuracast送信機の位置と情報を表示 |

### 発明の特徴的プロセス
1. クライアント端末がAuracast送信機を検出
2. 端末がAuracast情報（位置、コンテンツ種別、メタデータ）をサーバに送信
3. サーバがブロードキャスト情報を蓄積・管理
4. 別のクライアント端末が地図アプリでサーバからAuracast情報を取得
5. 地図上にAuracast送信機の位置とコンテンツ情報を表示

## 2. 最も近い先行技術との比較

### US 12,476,725 B2 との比較

| 項目 | US 12,476,725 B2 | 本発明 |
|------|-----------------|--------|
| **対象信号** | FM放送信号・ネットワーク放送 | Auracast（Bluetooth LE Audio）ブロードキャスト |
| **目的** | 放送信号の切替制御、信号強度に基づく自動切替 | Auracast送信機の発見支援、コンテンツ情報の事前提供 |
| **地図レイヤー** | 放送信号強度レイヤー（カバレッジ範囲） | Auracast送信機のPOI（点）表示 |
| **収集情報** | 位置、信号強度、変調周波数 | 位置、コンテンツ種別、BAP/BASSメタデータ |
| **主要ユースケース** | 車載端末での放送切替 | 公共施設の音声案内発見、美術館ガイド等 |

### 技術的相違点

1. **通信プロトコルの相違**
   - 先行技術: FM放送（アナログ/デジタル）、IP網
   - 本発明: Bluetooth LE Audio（Auracast）、専用プロファイル（BAP/BASS）

2. **収集データの相違**
   - 先行技術: 信号強度中心（カバレッジマップ生成）
   - 本発明: コンテンツメタデータ中心（POI情報提供）

3. **目的の相違**
   - 先行技術: シームレスな信号切替
   - 本発明: Auracast送信機の発見可能性向上

## 3. 新規性の評価

### 評価基準
日本特許法第29条第1項に基づき、「公然知られた発明」「公然実施された発明」「頒布された刊行物に記載された発明」に該当しないか評価

### 構成要素ごとの新規性評価

| 構成要素 | 先行技術での存在 | 評価 |
|----------|-----------------|------|
| A. Auracast受信端末 | Bluetooth LE Audio規格として存在 | × 新規性なし |
| B. Auracast情報収集 | 個別には存在するが、サーバ送信の組み合わせは未確認 | △ 要検討 |
| C. サーバ蓄積システム | クラウドサービス一般技術として存在 | × 新規性なし |
| D. Auracastメタデータ格納DB | **特定の組み合わせとして未確認** | ○ 新規性あり |
| E. Auracast地図表示 | **特定の組み合わせとして未確認** | ○ 新規性あり |

### 新規性の結論
**新規性あり（条件付き）**

- 個々の構成要素は既存技術の組み合わせ
- ただし、**Auracast特有のメタデータ（BAP/BASS規格に基づく情報）をサーバに蓄積し、地図上に表示する**という特定の組み合わせは、単一の先行技術に開示されていない
- US 12,476,725 B2 はFM/ネットワーク放送を対象としており、Bluetooth LE Audio（Auracast）は対象外

## 4. 進歩性の評価

### 評価基準
日本特許法第29条第2項に基づき、「当業者が容易に発明できたか」を評価

### 進歩性の論点

#### 論点1: 技術分野の近接性
- FM放送の地図表示（US 12,476,725 B2）とBluetooth LE Audioは、いずれも「放送信号」を扱う
- 当業者がFM放送技術をBluetooth LE Audioに適用する動機付けがあるか？

**評価**: △（動機付けの可能性あり）
- 放送信号の発見・表示という共通課題に対して、技術を転用する動機付けが認められる可能性

#### 論点2: 技術的課題の相違
- FM放送: 広域カバレッジ、信号強度変動、切替制御
- Auracast: 近距離通信、POI発見、コンテンツ情報提供

**評価**: ○（課題が異なる）
- 解決すべき技術的課題が異なるため、単純な適用では本発明に至らない

#### 論点3: 顕著な効果
本発明により得られる効果:
1. ユーザがAuracast送信機を事前に発見できる
2. 現地に行く前にコンテンツ情報を確認できる
3. クラウドソーシングによりデータベースが拡充される

**評価**: △（効果の顕著性は要強化）

### 進歩性の結論
**進歩性は認められる可能性があるが、強化が必要**

- 技術分野の近接性から、単純組み合わせとされるリスクがある
- 以下の点を強調することで進歩性を強化可能

## 5. 進歩性強化のための推奨事項

### 推奨1: Auracast固有の技術要素の活用

Auracast（Bluetooth LE Audio）特有の技術要素を発明に組み込む:

| 技術要素 | 活用方法 |
|----------|----------|
| BAP (Broadcast Audio Profile) | ブロードキャスト設定情報の収集・表示 |
| BASS (Broadcast Audio Scan Service) | スキャン結果の効率的収集 |
| BIS (Broadcast Isochronous Stream) | ストリーム情報の可視化 |
| LC3コーデック情報 | 音質・ビットレート情報の提供 |

### 推奨2: 技術的課題の明確化

FM放送地図とは異なる技術的課題を明確に記載:

1. **近距離通信特有の課題**: Auracastは10-100m程度の範囲であり、事前に存在を知ることが困難
2. **メタデータの豊富さ**: Bluetooth規格に基づく詳細なメタデータ（コンテンツ名、言語、音質等）
3. **動的なPOI**: 一時的なイベントでの利用（展示会、スポーツイベント等）

### 推奨3: 独自機能の追加検討

以下の機能を追加することで進歩性を強化:

1. **リアルタイム可用性表示**: 現在アクティブなAuracast送信機のみ表示
2. **コンテンツカテゴリ分類**: 案内音声、音楽、多言語対応等のカテゴリ表示
3. **ユーザ評価・レビュー機能**: クラウドソーシングによる品質情報
4. **プッシュ通知**: 近くのAuracast送信機の自動通知

**△ 進歩性あり（強化推奨）**

- 単純組み合わせとされるリスクを回避するため、Auracast固有の技術要素の活用を推奨
