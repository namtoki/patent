## 発明の概要

### 発明の名称（案）
「Auracast受信機の能力に基づくBIGチャンネル選択方法およびシステム」

### 技術分野
Bluetooth LE Audio（Auracast）におけるブロードキャストオーディオ配信

### 解決しようとする課題
Auracast Assistant が Auracast Receiver の特性（オーディオ能力）を考慮せずにBIGチャンネルを選択しているため、Receiverが対応できないストリームに接続しようとしたり、最適でないストリームを選択したりする問題がある。

---

## 発明の構成要素

### 主要な技術的特徴

1. **Receiver能力情報の取得**
   - PACS（Published Audio Capabilities Service）からReceiverのオーディオ能力を取得
   - 対応コーデック（LC3パラメータ）
   - 対応チャンネル構成
   - 対応サンプリングレート/ビットレート

2. **BIG情報の解析**
   - BASE（Broadcast Audio Source Endpoint）からBIGパラメータを抽出
   - 各BISのコーデック構成
   - チャンネルアロケーション
   - プレゼンテーションディレイ

3. **能力マッチングアルゴリズム**
   - Receiver能力とBIGパラメータの適合性判定
   - 複数BIGが存在する場合の最適BIG選択ロジック
   - 優先度に基づく選択（品質優先/省電力優先等）

4. **動的な再選択**
   - Receiverの状態変化（バッテリー残量等）に応じた再評価
   - 新規BIGの検出時の再マッチング

## 先行技術との比較分析

### 比較対象1: US-20250374018-A1 (Google)

| 観点 | Google特許 | 本発明 |
|------|------------|--------|
| **情報源** | 受信品質データ（reception data） | 受信能力情報（capability data via PACS） |
| **タイミング** | 受信後のフィードバック | 接続前の事前評価 |
| **調整対象** | ブロードキャストテンプレート（パラメータ調整） | BIG選択（どのBIGに接続するか） |
| **目的** | 受信品質の改善 | 最適なBIGへの接続 |

#### 差異点の技術的意義
- Google特許は「受信後」の品質改善、本発明は「接続前」の最適選択
- Google特許は「送信側の調整」、本発明は「接続先の選択」
- 本発明はReceiverの「静的能力」を活用、Google特許は「動的な受信状態」を活用

### 比較対象2: US-12445777-B2 (Intel)

| 観点 | Intel特許 | 本発明 |
|------|------------|--------|
| **焦点** | フィードバックチャンネルの確立 | 能力に基づくBIG選択 |
| **役割** | 通信路の確立 | 選択アルゴリズム |
| **関係** | 補完的（フィードバック手段） | 活用側（能力情報の利用） |

## 新規性の評価

### 新規性あり：以下の点で先行技術と区別される

1. **PACSを活用した事前能力評価**
   - 先行技術：受信品質に基づく事後調整
   - 本発明：受信能力に基づく事前選択

2. **BIG選択アルゴリズム**
   - 先行技術：単一ブロードキャストのパラメータ調整
   - 本発明：複数BIGからの最適選択

3. **Receiver特性の包括的評価**
   - 先行技術：受信状態のみ
   - 本発明：コーデック能力、チャンネル構成、品質設定等の総合評価

### 新規性の根拠

| 構成要件 | 先行技術での開示 | 評価 |
|----------|------------------|------|
| PACS情報の取得 | BASS標準で規定されているが、BIG選択への活用は未開示 | 新規 |
| 能力-BIGマッチング | 開示なし | 新規 |
| 最適BIG選択ロジック | 開示なし | 新規 |
| 動的再選択 | 品質ベースのみ（能力ベースなし） | 新規 |

## 進歩性の評価

### 進歩性あり：以下の技術的効果により

1. **接続失敗の回避**
   - 従来：能力不適合なBIGへの接続試行 → 接続失敗
   - 本発明：事前評価により適合BIGのみ選択 → 確実な接続

2. **ユーザー体験の向上**
   - 従来：試行錯誤的な接続
   - 本発明：最適なBIGへの即座の接続

3. **リソース効率**
   - 従来：不適合BIGへの無駄な接続試行
   - 本発明：効率的なBIG選択

### 進歩性の論理構成

**課題**: Receiverの能力を考慮しないBIG選択による接続問題

**解決手段**: PACS情報を活用した能力ベースのBIG選択アルゴリズム

**効果**:
- 接続成功率の向上
- ユーザー体験の改善
- システムリソースの効率化

**非自明性**:
- Google特許（受信品質ベース）からの動機付けなし
- PACS情報をBIG選択に活用するという着想は自明でない
- 能力マッチングアルゴリズムの具体的構成は技術的工夫を要する

## クレーム構成案

### 独立クレーム（方法）

```
【請求項1】
Auracast Assistantにおけるブロードキャストオーディオストリームの選択方法であって、
(a) Auracast Receiverから、Published Audio Capabilities Service (PACS)を介して、
    前記Receiverのオーディオ能力情報を取得するステップと、
(b) 利用可能な少なくとも1つのBroadcast Isochronous Group (BIG)から、
    各BIGのオーディオパラメータ情報を取得するステップと、
(c) 前記Receiverのオーディオ能力情報と、前記各BIGのオーディオパラメータ情報を
    比較し、前記Receiverが対応可能なBIGを判定するステップと、
(d) 前記対応可能なBIGから、所定の選択基準に基づいて最適なBIGを選択するステップと、
(e) 前記選択されたBIGへの接続を前記Receiverに指示するステップと、
を含むことを特徴とする方法。
```

### 従属クレーム案

```
【請求項2】
前記オーディオ能力情報は、対応コーデック、対応サンプリングレート、
対応チャンネル構成、および対応ビットレートの少なくとも1つを含む、
請求項1に記載の方法。

【請求項3】
前記選択基準は、オーディオ品質優先、省電力優先、または
ユーザー設定に基づく優先度の少なくとも1つを含む、
請求項1に記載の方法。

【請求項4】
前記Receiverの状態変化を検出し、前記状態変化に応じて
前記最適なBIGを再選択するステップをさらに含む、
請求項1に記載の方法。
```
